# DECISIONS.md

- **D1** Stack: React + TypeScript + Vite, Vitest, native HTML5 drag-and-drop — most standard stack, no DnD dependency needed for a simple vertical list. Approved deps only: react, react-dom, typescript, vite, @vitejs/plugin-react, vitest; anything else needs explicit approval. — 2026-07-03 (spec review)
- **D2** Persistence via localStorage auto-save on every change — spec omitted storage; simplest correct option for a single-user PWA; export/import is the backup path. — 2026-07-03 (spec review)
- **D3** History `day` = stored `currentDay` set by Start New Day, not wall-clock date at completion — working past midnight must not split a manual day's log. — 2026-07-03 (spec review)
- **D4** Start New Day auto-collapses Done into History under the old day before resetting Today — avoids stale strikethrough tasks lingering into the new day. — 2026-07-03 (user choice)
- **D5** Undo completion allowed (Done→Today; subtask un-tick) until cleared to History — protects against misclicks. — 2026-07-03 (user choice)
- **D6** Manual Today→Master removal allowed; day-copies are discarded on removal — fixes planning misclicks without waiting for Start New Day. — 2026-07-03 (user choice)
- **D7** Subtasks keep creation order, no drag reorder in MVP — not in spec, keeps scope down. — 2026-07-03 (spec review)
- **D8** History UI is a minimal read-only toggle panel grouped by day, newest first — spec defined History data but no display. — 2026-07-03 (spec review)
- **D9** Day-copies get `isRecurring: false`; provenance lives in `sourceTaskId` — the copy is an ordinary Today task, keeps sorting and deletion rules simple. — 2026-07-03 (spec review)
- **D10** Architecture: pure TypeScript core (state.ts et al.) with thin React shell — makes every spec rule unit-testable without a browser; suits red/green TDD. — 2026-07-03 (spec review)
- **D11** `completeTask` throws on open subtasks (rather than silently no-op) — the parent gate is a business-rule violation like an empty title, so a hard error is clearer; the UI also disables the Complete button as a first line of defence. — 2026-07-03 (slice 3)
- **D12** Subtask ops guard the "done never holds an open subtask" invariant by no-op: `addSubtask` and `uncompleteSubtask` do nothing on a Done parent (empty title still throws, matching D11's title rule) — keeps the model always valid; the undo flow is Done→Today first, then un-tick. — 2026-07-03 (slice 4) *(SPEC §6.4 amended 2026-07-04 to record this — it previously promised un-tick until cleared to History.)*
- **D13** `setActive`/`setActiveSubtask` toggle and are guarded by no-op (unknown id, or item not in Today) rather than throwing — activation is a benign UI gesture, not a business-rule violation like an empty title, so silent no-op is friendlier; single-active is enforced by clearing all flags before setting. — 2026-07-04 (slice 5)
- **D14** `removeFromToday` clears the returning task's active flags (task + subtasks); auto-clear on complete/delete falls out of existing code (own-flag clear / removal) — upholds the "active ⇒ in Today" invariant without extra plumbing. — 2026-07-04 (slice 5)
- **D15** `startNewDay` reuses `clearDone` for step 1 (before advancing `currentDay`, so Done logs under the old day) and `clearAllActiveFlags` for step 4 — no duplicate logic; the five ordered effects compose from existing pure functions. UI gates the button behind a `window.confirm` since discarding day-copies is irreversible. — 2026-07-04 (slice 6)
- **D16** Export uses a versioned envelope `{ schemaVersion, state }` rather than a bare AppState — a future incompatible shape change can be detected/migrated on import; `importState` validates the envelope and reuses `storage.isAppState` (exported to avoid a duplicate validator) so garbage throws a clear Error instead of crashing. — 2026-07-04 (slice 7)
- **D17** PWA is hand-rolled (no Vite PWA plugin, honouring D1): static `manifest.webmanifest` + generated 192/512 PNG icons + a minimal cache-first `sw.js` (precache shell, runtime-cache visited assets) registered in `main.tsx` — smallest correct install/offline path with zero new dependencies. — 2026-07-04 (slice 7)
- **D18** Subtask tick/un-tick is guarded to parents in Today (core no-op elsewhere; Master checkboxes disabled in UI) — a master task is a template, so ticking it polluted every future day-copy as pre-completed; SPEC §6.4 amended. — 2026-07-04 (post-slice-7 code review)
- **D19** Day-copies reset inherited subtask completion to open on creation — a fresh occurrence starts with all subtasks open; SPEC §6.3 amended. — 2026-07-04 (post-slice-7 code review)
- **D20** A completed subtask cannot be set active, and `completeTask` clears its subtasks' active flags — upholds "active ⇒ open item in Today" through the Done transition; SPEC §6.8 amended. — 2026-07-04 (post-slice-7 code review)
- **D21** `isAppState` validates element shapes (tasks, subtasks, history entries), `importState` requires `schemaVersion` exactly 1, and both load and import sanitise invalid active flags by clearing all — corrupt or hand-edited data now gets a clear error or a safe repair, never a crash. — 2026-07-04 (post-slice-7 code review)
- **D22** Service worker v2: navigations network-first (falling back to cache), install precaches the shell plus asset URLs extracted from index.html, hashed `/assets/` served cache-first, activate prunes assets no longer referenced; registration is production-only at `BASE_URL` — fixes first-visit offline, the stale-forever update pin, and the dev-server cache trap. — 2026-07-04 (post-slice-7 code review)
- **D23** Edit/Delete exposed in every column per SPEC §6.2, but outside Master the edit form offers title + due date only (no recurring toggle) — flipping a Today/Done task to recurring mid-flight would contradict day-copy semantics (D9). — 2026-07-04 (post-slice-7 code review)
- **D24** `clearActive` export removed as dead code — the un-set gesture is the toggle inside `setActive`/`setActiveSubtask`; no production caller existed. — 2026-07-04 (post-slice-7 code review)
