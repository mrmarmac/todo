# DECISIONS.md

- **D1** Stack: React + TypeScript + Vite, Vitest, native HTML5 drag-and-drop ‚Äî most standard stack, no DnD dependency needed for a simple vertical list. Approved deps only: react, react-dom, typescript, vite, @vitejs/plugin-react, vitest; anything else needs explicit approval. ‚Äî 2026-07-03 (spec review)
- **D2** Persistence via localStorage auto-save on every change ‚Äî spec omitted storage; simplest correct option for a single-user PWA; export/import is the backup path. ‚Äî 2026-07-03 (spec review)
- **D3** History `day` = stored `currentDay` set by Start New Day, not wall-clock date at completion ‚Äî working past midnight must not split a manual day's log. ‚Äî 2026-07-03 (spec review)
- **D4** Start New Day auto-collapses Done into History under the old day before resetting Today ‚Äî avoids stale strikethrough tasks lingering into the new day. ‚Äî 2026-07-03 (user choice)
- **D5** Undo completion allowed (Done‚ÜíToday; subtask un-tick) until cleared to History ‚Äî protects against misclicks. ‚Äî 2026-07-03 (user choice)
- **D6** Manual Today‚ÜíMaster removal allowed; day-copies are discarded on removal ‚Äî fixes planning misclicks without waiting for Start New Day. ‚Äî 2026-07-03 (user choice)
- **D7** Subtasks keep creation order, no drag reorder in MVP ‚Äî not in spec, keeps scope down. ‚Äî 2026-07-03 (spec review)
- **D8** History UI is a minimal read-only toggle panel grouped by day, newest first ‚Äî spec defined History data but no display. ‚Äî 2026-07-03 (spec review)
- **D9** Day-copies get `isRecurring: false`; provenance lives in `sourceTaskId` ‚Äî the copy is an ordinary Today task, keeps sorting and deletion rules simple. ‚Äî 2026-07-03 (spec review)
- **D10** Architecture: pure TypeScript core (state.ts et al.) with thin React shell ‚Äî makes every spec rule unit-testable without a browser; suits red/green TDD. ‚Äî 2026-07-03 (spec review)
- **D11** `completeTask` throws on open subtasks (rather than silently no-op) ‚Äî the parent gate is a business-rule violation like an empty title, so a hard error is clearer; the UI also disables the Complete button as a first line of defence. ‚Äî 2026-07-03 (slice 3)
- **D12** Subtask ops guard the "done never holds an open subtask" invariant by no-op: `addSubtask` and `uncompleteSubtask` do nothing on a Done parent (empty title still throws, matching D11's title rule) ‚Äî keeps the model always valid; the undo flow is Done‚ÜíToday first, then un-tick. ‚Äî 2026-07-03 (slice 4) *(SPEC ¬ß6.4 amended 2026-07-04 to record this ‚Äî it previously promised un-tick until cleared to History.)*
- **D13** `setActive`/`setActiveSubtask` toggle and are guarded by no-op (unknown id, or item not in Today) rather than throwing ‚Äî activation is a benign UI gesture, not a business-rule violation like an empty title, so silent no-op is friendlier; single-active is enforced by clearing all flags before setting. ‚Äî 2026-07-04 (slice 5)
- **D14** `removeFromToday` clears the returning task's active flags (task + subtasks); auto-clear on complete/delete falls out of existing code (own-flag clear / removal) ‚Äî upholds the "active ‚áí in Today" invariant without extra plumbing. ‚Äî 2026-07-04 (slice 5)
- **D15** `startNewDay` reuses `clearDone` for step 1 (before advancing `currentDay`, so Done logs under the old day) and `clearAllActiveFlags` for step 4 ‚Äî no duplicate logic; the five ordered effects compose from existing pure functions. UI gates the button behind a `window.confirm` since discarding day-copies is irreversible. ‚Äî 2026-07-04 (slice 6)
- **D16** Export uses a versioned envelope `{ schemaVersion, state }` rather than a bare AppState ‚Äî a future incompatible shape change can be detected/migrated on import; `importState` validates the envelope and reuses `storage.isAppState` (exported to avoid a duplicate validator) so garbage throws a clear Error instead of crashing. ‚Äî 2026-07-04 (slice 7)
- **D17** PWA is hand-rolled (no Vite PWA plugin, honouring D1): static `manifest.webmanifest` + generated 192/512 PNG icons + a minimal cache-first `sw.js` (precache shell, runtime-cache visited assets) registered in `main.tsx` ‚Äî smallest correct install/offline path with zero new dependencies. ‚Äî 2026-07-04 (slice 7)
- **D18** Subtask tick/un-tick is guarded to parents in Today (core no-op elsewhere; Master checkboxes disabled in UI) ‚Äî a master task is a template, so ticking it polluted every future day-copy as pre-completed; SPEC ¬ß6.4 amended. ‚Äî 2026-07-04 (post-slice-7 code review)
- **D19** Day-copies reset inherited subtask completion to open on creation ‚Äî a fresh occurrence starts with all subtasks open; SPEC ¬ß6.3 amended. ‚Äî 2026-07-04 (post-slice-7 code review)
- **D20** A completed subtask cannot be set active, and `completeTask` clears its subtasks' active flags ‚Äî upholds "active ‚áí open item in Today" through the Done transition; SPEC ¬ß6.8 amended. ‚Äî 2026-07-04 (post-slice-7 code review)
- **D21** `isAppState` validates element shapes (tasks, subtasks, history entries), `importState` requires `schemaVersion` exactly 1, and both load and import sanitise invalid active flags by clearing all ‚Äî corrupt or hand-edited data now gets a clear error or a safe repair, never a crash. ‚Äî 2026-07-04 (post-slice-7 code review)
- **D22** Service worker v2: navigations network-first (falling back to cache), install precaches the shell plus asset URLs extracted from index.html, hashed `/assets/` served cache-first, activate prunes assets no longer referenced; registration is production-only at `BASE_URL` ‚Äî fixes first-visit offline, the stale-forever update pin, and the dev-server cache trap. ‚Äî 2026-07-04 (post-slice-7 code review)
- **D23** Edit/Delete exposed in every column per SPEC ¬ß6.2, but outside Master the edit form offers title + due date only (no recurring toggle) ‚Äî flipping a Today/Done task to recurring mid-flight would contradict day-copy semantics (D9). ‚Äî 2026-07-04 (post-slice-7 code review)
- **D24** `clearActive` export removed as dead code ‚Äî the un-set gesture is the toggle inside `setActive`/`setActiveSubtask`; no production caller existed. ‚Äî 2026-07-04 (post-slice-7 code review)
- **D25** Relative due-date labels anchor on the wall-clock day (`useWallClockDay` in App.tsx), not `state.currentDay` ‚Äî a due date's "today"/"tomorrow" should track the real calendar so labels stay correct when the app sits open past midnight without Start New Day. The header "Day:" and History logging (D3) keep using the manual `currentDay`; only the labels move. The hook re-syncs on a 60s interval plus visibility/focus changes to catch the midnight rollover on throttled background tabs. ‚Äî 2026-07-08 (user request)
- **D26** A task title that is a bare URL renders as a tight clickable pill of just its domain (`parseUrl` in core/urls.ts, `TaskTitle` shared component) ‚Äî a read-later list of pasted links stayed legible instead of full-width truncated rows. Only single-token http/https/`www.` titles with a real domain pill; anything with spaces stays prose. In Today the pill replaces the activate-toggle button, so URL tasks are set active via keyboard (space) rather than a title click. ‚Äî 2026-07-08 (user request)
- **D27** The Master add-task form is progressively disclosed: only the "New task‚Ä¶" field shows until the form gains focus (CSS `:focus-within`), then the due-date row unfolds ‚Äî keeps the default UI to one line. The row is a single centre-aligned flex line (Due ¬∑ quick chips ¬∑ date ¬∑ Recurring ¬∑ Add) to kill the vertical zigzag. Added one-click relative due chips (Today / +1d / +7d) that toggle the date, computed from the wall-clock `today` via `addDaysISO`. ‚Äî 2026-07-08 (user request)
- **D28** Due-date labels are colour-coded by urgency (`dueDateUrgency`): overdue (red) / today (amber) / soon ‚â§2 days (yellow) get a tinted pill, further-out stays muted; Done cards suppress the colour so completed items don't shout. Title bar (`.app__header` + theme-color meta) now tracks the light/dark theme via `--header-bg` instead of a fixed brand purple, so the in-app header and the PWA's OS title bar read as one bar. Task-card vertical padding tightened (0.6‚Üí0.3rem). Manifest kept `display: standalone` (+ `display_override`/`id`) ‚Äî the chrome-less app window is what hides the browser toolbar and its extension icons. ‚Äî 2026-07-08 (user request)
